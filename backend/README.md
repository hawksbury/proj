# Backend API

Node backend for the disaster AI dispatch demo.

## Run

```bash
npm start
```

The server runs at `http://127.0.0.1:4000`.

To enable the real LLM summary, set your OpenAI API key before starting:

```bash
export OPENAI_API_KEY="your_api_key_here"
export OPENAI_MODEL="gpt-5.2"
npm start
```

If `OPENAI_API_KEY` is not set, the backend uses a local mock summary so the demo still works offline.

## Routes

- `GET /health`: API health check.
- `POST /api/alert`: receive a live distress signal, compute composite priority score, and add to the live queue.
- `GET /api/alerts`: return the current live queue sorted by priority score (max 50 alerts).
- `POST /api/signals`: receive a device signal and create a timestamped signal event.
- `GET /api/signals/:signal_id`: look up the customer connected to a device signal.
- `GET /api/responders`: list responder profiles for frontend login.
- `POST /api/dispatch`: run the full ML dispatch flow.
- `GET /api/dispatch/:signal_id`: quick demo route for the full dispatch flow.

## File Responsibilities

- `server.js`: Starts the HTTP server, loads `.env`, applies CORS headers, parses JSON request bodies, maps URLs to route handlers, and returns JSON errors.
- `package.json`: Defines backend scripts. `npm start` runs `node server.js`.
- `.env.example`: Shows the environment variables needed for real OpenAI summaries.
- `routes/signalRoutes.js`: Handles signal lookup routes. It receives a `signal_id`, finds the matching customer, and can create a timestamped signal event.
- `routes/dispatchRoutes.js`: Handles full dispatch routes. It connects signal lookup, ML prediction, responder matching, and LLM summary into one API response.
- `services/dataService.js`: Reads CSV files from the root `data/` folder and parses them into JavaScript objects. It caches loaded data for faster repeated requests.
- `services/signalLookupService.js`: Owns device-to-customer matching. It validates `signal_id`, finds the customer, and builds the `received_at` signal event.
- `services/priorityService.js`: Runs `backend/ml/predict_dispatch.py` as a child process and parses the ML JSON output.
- `services/responderMatchingService.js`: Converts ML-ranked responders into dashboard-ready options and explains why each responder matched.
- `services/llmDispatchService.js`: Creates the final dispatch summary. With an API key, it calls OpenAI and requests strict JSON. Without an API key or after an API error, it returns a mock summary.

## Example Dispatch Request

```bash
curl -s "http://127.0.0.1:4000/api/dispatch/SCRBS-0001?incident_type=power_outage"
```

## Dispatch Flow

1. Receive `signal_id`.
2. Match the device id to a customer in `data/customers.csv`.
3. Create `received_at` timestamp.
4. Call the trained ML models in `backend/ml`.
5. Return priority score, escalation probability, responder profession, ranked responder options, and dashboard summary.

The dashboard summary is generated by `services/llmDispatchService.js`. With an API key, it calls the OpenAI Responses API with structured JSON output. Without an API key, it returns a deterministic mock summary.
